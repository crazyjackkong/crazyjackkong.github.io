git 笔记

发表于 2021-02-05 更新于 2023-03-21 分类于 学习 阅读次数： 本文字数： 1.5k 阅读时长 ≈ 1 分钟

参照以下链接进行在线交互式学习：
https://learngitbranching.js.org/?NODEMO=&locale=zh_CN

基础概念

  1. git 三个区：
    工作区working dirctory—>暂存区staged / index –> 版本库repository

基本操作

  1. 文件提交
    working–>>staged :git add <file>
    staged –>>repository:git commit -m "<message>
    working–>>repository:git commit -a

  2. 文件查看区别

    1. working 和 satged :git diff

    2. 和 repository ：git diff --staged

  3. 文件回退

    1. working dirctory回退至staged内容:git checkout -- <file>
      staged 回退至repository内容:git reset HEAD <file>

    2. working 回退至repository内容：git reset HEAD <file>再git checkout -- <file>

    3. repository回退上一个版本：git reset --hard HEAD^；上两个版本git reset --hard HEAD^^

    上n个版本：git reset --hard HEAD~<n>

    1. 回退至指定版本：git reset --hard HEAD <commit id>

    2. 将文件从staged移除：git rm <file>

    3. 将文件从repository移除：git rm <file>后commit

    4. 为了撤销更改并分享给别人，我们需要使用git revert，相当于一次新的提交，用于抵消要撤销的提交

  4. 分支操作

    1. 合并分支: 一般有两种方法：rebase、merge，但一般用rebase
      git rebase <前面的分支> <后面的分支>
      而且 git rebase -i HEAD~4可以进行交互式操作之前的相应数量的提交

    cherry-pick 移动指定提交: git cherry-pick <提交1> <提交2> 将提交移动到当前的HEAD之后

  5. 其他

    1. 查看git工作状态:git status

    2. 查看log文件：git log --pretty=oneline

    3. 查看历史提交信息（用于版本恢复）：git reflog

    可参考链接：https://www.jianshu.com/p/e1a981eec3d3?utm_campaign=maleskine&utm_content=note&utm_medium=seo_notes&utm_source=recommendation

  6. 远程操作

    1. 链接远程库：git remote add origin git@github.com:crazyjackkong/<repository>.git
    2. 推送：git push origin master可以在push 后加-u参数用于关联
    3. 拉取：

git fetch 完成了仅有的但是很重要的两步:
从远程仓库下载本地仓库中缺失的提交记录
更新远程分支指针(如 origin/main)
git fetch 并不会改变你本地仓库的状态。而pull 却可以。
git pull 就是 git fetch 和 git merge 的缩写！同时完成了下载和合并本地分支的作用
git pull --rebase 就是 git fetch 和 git rebase 的简写！
